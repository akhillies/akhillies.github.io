Shading

Here is the second project I worked on. This one focuses on shading and lighting - understanding the math and the ideas behind each aspect of a still picture. I did everything in this project without to use of standard libraries like OpenGL in order to gain a better understanding of what is going on behind the scenes. I had to draw 3D shapes combining different types of lights (directional, point, etc) and different types of shading (diffuse, specular, etc.). This actually served as a basis for future projects as while I never coded another shader, it helped me understand why something turned out a certain color as well as how to determine what kind of lights to use for different situations. I will be happy to answer any more questions you may have: just email me at akhilbatra@berkeley.edu.

Personal Comments / Response

This project was another interesting assignment for me. I understood all the math behind the shading fairly quickly, and despite that it still took awhile to understand what exactly knowing the theory behind shading meant for code. Eventually, when I reached the end of the tunnel so to speak, I realized the only thing needed to code shading was to understand the math - there were no tricks or complicated procedures involved. Breaking past this barrier of there having to be some detailed template or set of instructions was honestly the hardest part of this project. It was the first time that I had much more freedom than I was used to in a school assignment (the last project was fairly simple and did not really need any complicated programs). Another problem that came up had to do with outputting the information into a readable image file (like PNG or JPEG). This is the only thing I had planned to complete before the deadline but was unable to; the interesting part was I was so close to the end, I just needed to change how the pixels were written to the file using FreeImage and it would've worked (as it did for my next project). After this project, I am much more proficient in reading code to find bugs as well as much more comfortable in writing a program an infinite scope for what is possible.

Specular Shading and Directional Light

./as1 -pl 200 200 200 0.6 0.6 0.6 -ks 0.8 0.8 0.8 -sp 16
Legend of tags used:

-pl x y z r g b

Creates a Point Light at position (x, y, z) with the passed RGB values

-dl x y z r g b

Creates a Directional Light at position (x, y, z) with the passed RGB values

-kd r g b

Uses the passed RGB as the Diffuse Shading constant for each object

-ka r g b

Uses the passed RGB for the Ambient Shading constant of each object

-ks r g b

Uses the passed RGB as the Specular Shading constant of each object (must be accompanied by the -sp tag)

-sp i

Uses the Intensity for the calculation of Specular Shading for each object (must have a specular shading tag defined as well)

-cn

Add if you want all the objects to be given a cartoon-ish look

-ns n

Tag to specify number of spheres (n). Defaults to 1, so 0 must be specified

-nc n

Tag for specifying the number of cubes wanted in the image. Defaults to 0
